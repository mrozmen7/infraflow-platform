# Agentic Engineering Terimler Sözlüğü

| Terim | Türkçe anlamı | Kısa açıklama |
|---|---|---|
| Acceptance Criteria | Kabul kriterleri | Tek özelliğin doğru sayılması için test edilebilir sonuçlar |
| ADR | Mimari karar kaydı | Kalıcı kararın bağlam, alternatif, karar ve bedel kaydı |
| Agent | Amaç doğrultusunda araç kullanabilen yapay zekâ yürütücüsü | Yalnız sohbet değil; okuyabilir, planlayabilir, değiştirebilir ve doğrulayabilir |
| Approval | İnsan onayı | Sınır aşan veya yan etkili işleme verilen dar kapsamlı izin |
| Assumption | Varsayım | Henüz kanıtlanmamış kabul |
| A2UI | Agent-to-UI standardı | Ajanın yapılandırılmış ve render edilebilir UI tanımı iletmesi |
| AG-UI | Agent–User Interaction protokolü | Agent olayları/state/tool sonuçları için gevşek bağlı iletişim modeli |
| Cache | Önbellek | Tekrar kullanılmak üzere geçici saklanan sonuç |
| Characterization Test | Mevcut davranışı tanımlayan test | Refactor öncesi bug/risk görünürlüğü sağlar |
| CI | Sürekli entegrasyon | Clean ortamda otomatik test/build doğrulaması |
| Context | Ajanın aktif bağlamı | Karar verirken görebildiği talimat, dosya ve sonuçlar |
| Contract | Sözleşme | İki sınırın veri/davranış anlaşması |
| Definition of Done | Tamamlanma tanımı | Bütün teslimatların ortak kalite koşulları |
| Deterministic | Belirlenimci | Aynı girdide aynı sonucu üreten |
| Diff Review | Değişiklik farkı incelemesi | Sadece final dosyayı değil neyin değiştiğini değerlendirme |
| E2E | Uçtan uca test | Gerçek kullanıcı akışını birçok katmanla birlikte doğrulama |
| Evidence | Kanıt | Teknik iddiayı destekleyen tekrarlanabilir sonuç |
| Eval | Ajan değerlendirmesi | Agent davranışını ölçülebilir kriterlerle puanlama |
| Feature Flag | Özellik bayrağı | Davranışı deploy'dan bağımsız kontrollü açıp kapatma |
| Golden Master | Altın ana çıktı | Karmaşık mevcut sistem çıktısını geniş biçimde koruyan referans |
| Guardrail | Koruma kuralı | Yasak veya zorunlu davranışı otomatik kontrol eden sınır |
| Human-in-the-Loop | Döngüde insan | Kritik kararın insan onayı olmadan tamamlanmaması |
| Idempotent | Tekrarda aynı etkiyi üreten | Aynı komutun güvenli tekrarını tanımlar |
| Legacy Code | Miras/eski kod | Değiştirilmesi riskli, davranış kanıtı zayıf kod |
| MCP | Model Context Protocol | Modelin araç/veri kaynaklarına standart sözleşmeyle bağlanması |
| Non-goal | Bilinçli kapsam dışı hedef | Bu teslimatta yapılmayacağı açıkça belirtilen konu |
| Optimistic Update | İyimser güncelleme | Server cevabından önce UI state'ini beklenen sonuca alma |
| Port | Soyut bağımlılık sözleşmesi | Domain/application'ın concrete adapter yerine kullandığı arayüz |
| Prompt Injection | Talimat enjeksiyonu | Güvenilmeyen içeriğin ajanı asıl görevden saptırma girişimi |
| Quality Gate | Kalite kapısı | Teslim için geçmesi gereken zorunlu kontrol zinciri |
| Refactoring | Davranışı koruyarak iç tasarımı iyileştirme | Feature eklemekten farklıdır |
| Regression | Geri gelen/oluşan bozulma | Önceden çalışan davranışın yeni değişiklikle kırılması |
| Rollback | Geri alma | Başarısız değişiklik veya state'i güvenli önceki hâle döndürme |
| Sandbox | İzole teknik çalışma sınırı | Dosya/network erişiminin teknik kısıtı |
| Scope | Kapsam | Görevin yapacağı ve yapmayacağı işler |
| Seam | Ayırma dikişi | Büyük rewrite olmadan bağımlılığı ayırma/test etme noktası |
| Signal | Reaktif değer taşıyıcısı | Angular'ın değer değişimini tüketicilere bildiren state primitive'i |
| Specification | Özellik tanımı | Problem, kapsam, kural, kalite ve risk belgesi |
| Stop Criteria | Durma koşulları | Agent'ın insan kararı istemesi gereken önceden tanımlı durumlar |
| Subagent | Alt ajan | Ana görevin sınırlı ve bağımsız parçasını yürüten yardımcı ajan |
| TDD | Test güdümlü geliştirme | Red → Green → Refactor döngüsü |
| Threat Model | Tehdit modeli | Varlık, saldırgan, trust boundary ve savunma analizi |
| TTL | Önbellek yaşam süresi | Cache kaydının taze sayıldığı süre |
| Untrusted Input | Güvenilmeyen girdi | Yetkili talimat sayılmayan dış içerik/veri |
| Worktree | Ayrı Git çalışma ağacı | Aynı repository için ayrı fiziksel checkout |
